# Bitlog

Key-value хранилище на базе механизма Bitcask и совместимое с протоколом Redis.

## Цели

Основная цель заключается в том, чтобы превзойти Redis в операциях записи, обновления и удаления.

## Структура

- **Bitcask** - механизм хранения данных
- **Network** - сетевое взаимодействие

### Bitcask

В Bitcask существует два основных компонента: KeyDir и DataFile.

- KeyDir - это по сути, представление текущего состояния БД в памяти. Оно содержит информацию о ключах и о том в каком файле находится соответствующее значение. На этапе MVP будет использоваться HashTable с мьютексом, однако в будущих версиях планируется использование SkipList.

- DataFile - бинарный файл с доступными операциями: write ahead и read. После каждого рестарта БД будет создаваться новый файл.

### Сетевое взаимодействие

- Через TCP-соединение с использованием протокола Redis
