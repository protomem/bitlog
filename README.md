# Bitlog

Key-value хранилище на базе механизма Bitcask и совместимое с протоколом Redis.

## Цели

Основная цель это сравняться или превзойти Redis в операциях write, update и delete.

## Структура

- **Bitcask** - механизм хранения данных
- **Network** - сетевое взаимодействие

### Bitcask

В Bitcask существует два основных компонента: KeyDir и DataFile.

- KeyDir - это по сути, представление текущего состояния БД в памяти. Оно содержит информацию о ключах и о том в каком файле находится соответствующее значение. На этапе MVP будет использоваться HashTable с мьютексом, но в будущих версиях запланировано использование SkipList.

- DataFile - бинарный файл с доступными операциями: write ahead и read. После каждого рестарта БД будет создаваться новый файл.

### Сетевое взаимодействие

- По TCP-соединению и протоколу Redis
